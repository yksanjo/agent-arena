## Problem Statement
Yoshi (@yksanjo) wants a Polymarket-style platform where AI agents bet against each
other on specific events. Chosen framing (from office-hours): **agents DUEL on a
prediction**. Two named agents make falsifiable, timestamped forecasts on a defined
claim; the market resolves by which agent was more correct. The product is a public
arena / leaderboard of agent predictive skill. Builds directly on the existing
16-agent SOAG grid. He says he can't implement it himself but believes it has large
potential, and asked first whether it's feasible.

## What Makes This Cool
The asset being minted is an **ungameable track record of predictive skill**. Humans
can't produce this — they cherry-pick, delete bad calls, never timestamp. An agent
posting signed, timestamped, auto-resolving predictions on-chain accrues a credential
a human literally cannot fake. The leaderboard is a credential, the bet is downstream
of the credential, and the credential is the moat. It is also a falsification harness,
which matches Yoshi's core epistemic pattern (he keeps honestly proving things −EV).

## Constraints
- Paper / $SOAG first. Real human money on outcomes = prediction market = CFTC/gambling
  surface in the US. The benchmark version has zero regulatory surface.
- $0 / near-$0 infra (free price APIs, edge caching, Pi or Cloudflare Worker), matching
  his conservation-mode discipline.
- Must reuse the SOAG grid agents rather than building a new fleet.

## Premises (agreed framing)
1. **Auto-resolvable claims only — no free text.** Restrict to claims an oracle/price
   feed resolves with zero human judgment ("token up/down in 24h"). This is the single
   biggest feasibility unlock — it sidesteps Polymarket's resolution/dispute problem.
2. **The forecasting side is never empty.** The grid agents forecast 24/7 for free, so
   the leaderboard populates with zero humans present. The human-betting layer is LAYER
   2, bolted on once the leaderboard is interesting. Do not gate the arena's existence
   on having gamblers.
3. **"Agents" is the product, not a gimmick.** The novel, defensible thing is the
   ungameable forecast track record.
4. **Score calibration (Brier / log-loss), not alpha.** If ranked by returns, a no-edge
   world makes the board worthless. Ranked by calibration, even "no edge exists" is a
   true, rankable, publishable result. This protects the idea from Yoshi's own −EV thesis.
5. **Who-pays is an honest open question.** Near-term most likely value is
   benchmark-as-marketing for the SOAG brand + grid. Revenue bets, least to most
   speculative: copy proven-edge agents (human sub) → x402 charge-the-agent for
   entry/query → proof-of-forecast credential other agents pay to read.

## Approaches Considered

### Approach A: The Arena benchmark (minimal, paper, no money)
Grid agents emit timestamped confidence forecasts on ONE auto-resolvable claim type
(token up/down in 24h across a basket). Resolver cron pulls price feed, scores Brier +
calibration, updates an ELO-style board. One public page: leaderboard + per-agent
calibration curve + shareable prediction cards (reuse the SURVIVORS /t/:mint atom).
- Effort: S/M  Risk: Low  Reuses: SOAG grid, free RPC/price API, edge cache, OG-card pattern

### Approach B: Agent Arena with $SOAG stakes + duels + on-chain proof (ideal)
Agents post signed predictions on-chain (Solana memo/PDA) for an ungameable record.
Head-to-head duels: two agents, one claim, $SOAG staked into escrow, auto-resolved from
oracle, winner takes stake; parimutuel pool for humans backing a duelist. x402 entry/query
fees.
- Effort: L  Risk: Med (escrow write-path is the untested piece flagged on soag-lock;
  needs real signed-tx testing before money moves). Keep human real-money betting gated
  behind $SOAG/paper to stay clear of the regulatory line.
- Reuses: soag-lock escrow patterns, x402/gmem-paywall, Token-2022 $SOAG verify path

### Approach C: Proof-of-Forecast credential primitive (lateral / north star)
Not a destination — a primitive/SDK where ANY agent posts a signed, auto-resolving
prediction and accrues portable on-chain forecast reputation. Run the first leaderboard
as the reference client. Bet: in an agent economy, "can this agent predict?" becomes a
thing agents must prove to transact; you own the rails, x402-native.
- Effort: M+ ongoing  Risk: High (infra bet on agent-economy adoption; ghost-town risk)

## Recommended Approach
**A now, built so B bolts straight on, with C as the framing for why B matters.**
Ship the paper benchmark first. It's cheap, honest, produces a true result whether or
not edge exists, has zero regulatory/infra cost, and is the atom everything else
attaches to. Only build escrow/stakes (B) once the leaderboard is interesting enough
that someone wants to bet on it.

## Open Questions
- Which claim type for v1? (token up/down 24h is the cleanest; could also do
  "which of two tokens outperforms over 24h" = a literal duel with a clean binary.)
- Are the grid agents actually *differentiated* on calibration, or do they all converge
  to ~0.25 Brier (noise)? If undifferentiated, there is no arena worth betting on. This
  is the kill-or-continue test and it costs one weekend.
- Resolution source: on-chain price (Pyth/oracle) vs free REST (DexScreener/Gecko). REST
  is fine for the paper benchmark; on-chain oracle needed before staked duels.

## Success Criteria
- v1 (Approach A): N grid agents produce ≥1 week of timestamped forecasts on one claim
  type, auto-scored by Brier, rendered on one public page with shareable cards. Pass =
  the agents are *measurably differentiated* in calibration (not all noise).
- Only then consider B.

## Distribution Plan
- v1: one public page on existing infra (Cloudflare Worker like degenscreener, or Pi +
  Caddy). Shareable per-prediction OG cards are the distribution atom (proven pattern
  from SURVIVORS / soag desk). Lead any announcement with a meme per standing rule.
- B: x402 endpoints for agent-to-agent read/write; human copy-subscription as first
  dollar.

## Next Steps
1. Pick ONE auto-resolvable claim type (recommend "token up in 24h" or the two-token
   duel binary).
2. Add a forecast emitter to the grid agents: {agent_id, claim_id, p(YES), ts, signature}.
3. Write the resolver cron: fetch outcome, compute Brier/log-loss, update calibration +

4. One page: leaderboard + calibration curves + shareable prediction cards. No betting.
5. Run a week. Check differentiation. Decide on B from real data.
